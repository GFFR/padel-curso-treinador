# Question Reporting

Students can give quick thumbs up/down feedback on a question and can open a support message when something seems wrong. The support bubble should support both general messages from anywhere in the app and question-specific reports when opened from a question screen.

Every question report should capture:

- Stable question ID
- Prompt
- Four answer options
- Correct answer
- Explanation
- Source references
- Course theme
- Source scope
- Exam attempt ID
- Student selected answer, when available
- Student message

This keeps reports useful even if the question is later edited, regenerated, or removed from the active question bank.

General support messages do not need question context, but should still capture the student account when available and whether the message is a suggestion or bug report.
